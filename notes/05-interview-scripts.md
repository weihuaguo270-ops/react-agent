# 面试话术 — Agent 交互能力优化

> 以下两个经历可以直接回答面试官关于"如何提升 Agent 产品的交互体验"的问题。

---

## 经历 1：首次使用引导（`agent --setup`）

### 适合回答的问题

- "你怎么让新用户快速上手你的 Agent 产品？"
- "你在项目中做过哪些用户体验相关的优化？"
- "第一次运行你的 Agent 是什么体验？"

### 话术

> 最初用户 clone 项目后，需要自己去找 `llm_config.json` 文件、查文档才知道怎么填 API Key。我做的优化是加了一个 `agent --setup` 交互式配置向导。
>
> 用户装完项目后，只需要在终端输入 `agent --setup`，就会进入一个菜单——选择 LLM 提供商、输入 API Key、选模型，全程只需要不到 30 秒。配置完后直接 `agent "你的问题"` 就能用。
>
> 同时也在 `--help` 里展示了所有可用命令和环境变量，用户不用翻文档就能上手。

### 要点总结

| 之前 | 之后 |
|------|------|
| 用户需要翻文档 → 找 `llm_config.json` → 编辑 JSON | 终端输入 `agent --setup` → 菜单引导 → 30 秒配好 |
| `--help` 只显示 4 行基本命令 | 补充了 `--mcp`、`--parallel`、环境变量提示 |

## 经历 2：加速 Agent 响应速度（懒加载）

### 适合回答的问题

- "你怎么提升 Agent 的响应速度？"
- "你的项目在启动时有什么优化？"
- "你在项目中做过什么性能优化？"

### 话术

> 项目依赖 BGE 语义模型来做记忆和文档检索，每个模型 ~33MB。之前每次运行 `agent` 都要加载模型 + 索引项目文件，光是启动就要等 15 秒，连 `--help` 也要等。
>
> 我的做法是把模型加载从"启动时立即加载"改成"首次使用时懒加载"：
> - `agent --help` 和 `agent --setup` 不需要模型，完全跳过加载
> - 用户第一次提问时才加载模型，后续复用
>
> 结果：`agent --help` 从 15 秒缩短到 1.6 秒，`--setup` 也基本秒开。用户第一次问问题时总耗时不变（因为无论如何都需要加载一次模型），但日常使用中的启动体验大幅提升。

### 要点总结

| 之前 | 之后 |
|------|------|
| 所有命令（包括 `--help`）都要等 ~15 秒模型加载 | `--help`、`--setup` 秒开，~1.6s |
| BGE 模型在 `__init__` 时立即加载 | `_get_model()` 按需加载，首次使用时才初始化 |
| RAG 索引在 import 时自动执行 | `_ensure_rag_loaded()` 只在进入 `react_loop()` 时触发 |
