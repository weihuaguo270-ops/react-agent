# Dashboard 开发心路历程

## 最初想实现的功能

1. **轨迹回放**：Agent 跑完后，能在 Web 页面上像翻书一样逐步骤查看它的决策过程（thought → action → observation → final answer）
2. **交互式执行**：用户可以在界面上直接输入问题，Agent 实时执行，并同时显示聊天记录和轨迹回放

## 遇到的主要问题

### 问题 1：第一版界面太简陋

第一个版本是一个全屏的简单卡片列表，轨迹文件名用按钮堆在一起，没有搜索、没有导航。

面试级展示时给人感觉像 demo 不是产品。

### 问题 2：后端子进程是无状态的

`/api/chat` 接口每次通过 `subprocess.run()` 调起 `react_loop.py`，Agent 的状态不能跨请求保持——每次 query 都是独立会话。

然而轨迹查看器需要连续查看同一 session 的多步决策过程。

### 问题 3：老代码修改后服务未重启造成假阴性

由于之前启动的 server.py 进程还在运行，修改后新代码没生效，花了较多时间排查。

## 优化方案

### 第一轮：布局重构

从全屏列表改成**左侧面板 + 右侧主区**布局：

- 左侧：可滚动的轨迹列表 + 搜索框（按 query 过滤）
- 右侧：统计面板（步数/耗时/Token）+ 折叠式步骤卡片
- 支持键盘快捷键（← → 切换步骤，空格 播放/暂停）

### 第二轮：对比模式

用户可在左侧勾选多条轨迹，右侧同时展示统计摘要和最终答案，方便对比不同配置下的表现。

### 第三轮：聊天 + 轨迹面板合一

后台新增 `/api/chat` 接口，前端左侧加聊天面板：

1. 用户在左侧输入 query
2. 后端 subprocess 调起 react_loop.py 执行
3. 返回 stdout 和最新轨迹 JSON
4. 左侧显示聊天记录
5. 右侧自动加载新轨迹并回放

### 额外：Agent 工具调用启动 Dashboard

在 Agent 的工具列表里注册了 `start_dashboard` 工具，用户对 Agent 说"打开 Dashboard"即可自动启动服务，不需要记命令。

## 当前状态

- 访问 `http://127.0.0.1:5050` 即可使用
- 同时支持历史轨迹回放和实时执行
- 可访问轨迹目录下的全部历史文件（不丢失数据）
